The Flaky Network: architecture and design

Sender

The sender receives one numbered 160-byte frame every 20 ms. It divides the frame immediately into four 40-byte data shards and creates three 40-byte parity shards using a Cauchy coding matrix over GF(256). All seven shards are sent as soon as that frame arrives, so repair information never waits for a later source frame.

Wire format and bandwidth

Every sender-to-receiver datagram contains a 4-byte big-endian block number, a 1-byte shard index, and a 40-byte shard. Seven 45-byte packets protect one 160-byte frame: 7 * 45 / 160 = 1.96875x. No feedback packets are used, leaving the measured overhead safely below the 2.0x limit.

Receiver

The receiver stores unique shards by block and ignores relay duplicates. Four systematic shards can be joined directly into the original frame. If any are lost, receipt of any four of the seven total shards supplies four independent equations. Gaussian elimination in GF(256) reconstructs the missing pieces byte-for-byte, after which the receiver emits the 160-byte frame with its original sequence number.

Why forward error correction

Retransmission requires a loss to be noticed and then sends both a request and a reply through the same lossy, delayed relay. Proactive parity spends bandwidth before a loss is known and usually makes recovery available as soon as the last data/parity packets in a block arrive. Up to three lost datagrams in one seven-packet block are recoverable.
